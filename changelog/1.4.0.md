# 1.4.0 — 2026-04-25

> Soul, multi-key BYOK, self-update, models refresh, TS extensions API, `dirgha kb`, skill-injection scanner, parallel-fleet audit, and 17 real bugs fixed. **40/40 offline tests in 16s. Parity matrix sum-of-gaps = 0.**

## Highlights

- **Persona** — soul Markdown at `~/.dirgha/soul.md`, ships with a sane default. Defines tone, boundaries, end-of-turn norms.
- **BYOK pool** — multi-key per provider with priority + LRU + cooldown rotation across **17 known providers**. Atomic file lock; mode 0600.
- **Self-update** — `dirgha update --check / --self / --packages [--yes]`. npm-registry-aware, prompt-gated, audit-logged.
- **Auto-fetch catalogue** — `dirgha models refresh` parallel `/v1/models` fetch + 24h cache. Live: 499 models in <1s.
- **Plugins** — TypeScript / ESM extensions API at `~/.dirgha/extensions/<name>/index.mjs`. `registerTool / registerSlash / registerSubcommand / on(event)`.
- **Knowledge base** — `dirgha kb` wraps OpenKB + PageIndex for vectorless reasoning-based RAG. `~/.dirgha/kb/`.
- **Skill safety** — heuristic scanner runs at install + load. Critical findings block. `dirgha skills audit` re-scans on demand. Live result on 112 installed skills: **74 allow / 36 warn / 2 block**.
- **Parallel audit** — `dirgha audit-codebase` fans sub-agents across every src module via the fleet primitive, synthesises one report.

### Fixed

- **NIM streaming silently dropped content** on `deepseek-v4-flash` because the parser only recognised `delta.reasoning_content`, not `delta.reasoning`. Both keys are now accepted.
- **NIM + OR per-call timeouts** were 60 s / 120 s — too tight for multi-turn reasoning models. Bumped to 300 s.
- **Mid-session failover** previously triggered only on turn 0. Now resumes from `result.messages` with `maxTurns − turnCount` budget remaining.
- **`dirgha keys set <ENV>` never reached providers** — keystore wasn't hydrated into `process.env` at startup. Fixed; shell env still wins.
- **Login subcommand flags were swallowed** by the top-level parser. Now passed through verbatim.
- **`task` tool was implemented but never registered** with the runtime tool registry. Wired in `main.ts` via SubagentDelegator.
- **ErrorClassifier was implemented but never instantiated.** Now wired into `runAgentLoop` in all three entry points (main / interactive / Ink).
- **`skills install <url>` flag injection** — defensively reject URLs starting with `-`.
- **Test paths hardcoded** to absolute prefixes — now portable via `import.meta.url`.

### Internal

- 6 new src modules: `extensions/`, `security/`, `auth/keypool.ts`, `auth/providers.ts`, `context/soul.ts`, `intelligence/models-refresh.ts`.
- 11 new offline test suites (keypool, soul, update, models-refresh, extensions, kb, skill-scan, …). Total **40/40 green in ~16 s**.
- 2 high-severity findings from the codebase audit fixed: skills-install URL flag-prefix rejection + portable test paths.
- Soul now references the operating norms in [`docs/agents/files-and-search.md`](../docs/agents/files-and-search.md), [`docs/agents/skill-security.md`](../docs/agents/skill-security.md), and [`docs/DOCS-CONVENTION.md`](../docs/DOCS-CONVENTION.md).

## Migration

No breaking API changes. New surfaces are opt-in:

- The skill scanner runs by default. To install a skill that the scanner blocks (e.g. a deploy skill that legitimately says "do NOT ask for confirmation"), pass `--force-unsafe-install` and review the body manually first.
- Existing `~/.dirgha/keys.json` (legacy single-slot) still works. Pool entries are layered ON TOP — pool wins when both define the same env name.
